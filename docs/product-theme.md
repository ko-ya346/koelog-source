# KOELOG product theme

## 一言でいうと

KOELOG は、音声または文字入力で日々の記録を残し、ユーザーの行動を定量化して振り返りと次の行動につなげる AI 秘書。

## 目指す体験

- 音声または文字で、その日の仕事、作業、健康、食事、支払い、締切、目標進捗を記録できる。
- 入力された自然文から、何をしたか、どれくらい時間を使ったか、何を作ったか、目標に対してどれだけ進んだかを構造化できる。
- 1日、1週間、1ヶ月単位で行動を振り返り、次に取るべき行動のアドバイスを受けられる。
- 支払い期限や制作締切などを抽出し、リマインドできる。
- 複数人で記録や進捗を共有できる。
- 開発中も実機で手触りを確認できるよう、早い段階でデプロイ方法を確立する。

## 現在の実装との差分

現在は単一ページのフロントエンド試作で、音声入力、簡易カテゴリ判定、ローカル保存、固定のダッシュボード表示がある。

不足している主な領域:

- ユーザー、チーム、共有権限の設計
- 永続 DB と API
- AI による自然文解析、構造化、月次振り返り
- 行動時間、成果物、目標進捗の定量モデル
- 締切、支払い期限、通知、リマインド
- 複数人共有 UI
- デプロイ、環境変数、運用確認の手順
- 現実の画面に合ったテスト

## 初期開発の優先方針

1. デプロイ導線を早めに確立し、スマホや実環境で触れる状態にする。
2. localStorage 前提をやめ、Vercel で運用しやすい永続 DB に記録を保存する。
3. 記録データを構造化し、行動時間、カテゴリ、成果物、目標進捗、期限を扱える形にする。
4. AI 解析と月次振り返りは、まず手動入力でも検証できる最小仕様から始める。
5. 複数人共有は、ユーザー/チーム/権限モデルを先に決めてから UI を広げる。
